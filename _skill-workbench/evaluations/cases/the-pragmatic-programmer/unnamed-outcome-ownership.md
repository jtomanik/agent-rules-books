# Own the Export Outcome

The export team has been told to ship in ten days and to follow the existing
style. That style copies the same eligibility calculation into the API, nightly
job, and admin preview. The new vendor format is written directly into domain
records. Releases require one engineer to run six undocumented commands and
compare output by eye.

The delivery manager says each issue belongs to a different team or tool:

- duplicated rules belong to the owners of each entry point;
- vendor lock-in is the framework's default;
- the manual release is acceptable because the release engineer knows it;
- architecture feedback can wait until all three entry points are complete.

Review that position and propose a thin, reviewable first slice. Identify who
must own the result, which knowledge needs one authority, which commitment
should remain reversible, and how feedback should arrive before the deadline.
