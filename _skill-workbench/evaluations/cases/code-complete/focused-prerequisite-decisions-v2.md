# Resolve Construction Prerequisites Before an Irreversible Integration

A team is about to build a large integration against a proprietary document SDK. Requirements cover the happy path, but error policy, data representation, reuse boundaries, integration shape, and test strategy are unsettled. The vendor contract is expensive to reverse after rollout.

One reviewer says implementation should start and let details emerge. Another says the source's construction-prerequisite guidance requires specific decisions first, while still using a small validated slice where uncertainty remains.

Resolve this bounded interpretation dispute. Name the exact construction decisions that must be clear enough before large implementation work, distinguish reversible uncertainty from expensive commitments, and define the smallest validated slice that can reduce remaining risk. Do not design the entire integration.
