# Run a Complete DDIA Assessment

Use `$designing-data-intensive-applications` and apply its complete reference end to end.

Audit an order platform with PostgreSQL primary writes, asynchronous replicas, Redis caches, Kafka commands/events, at-least-once consumers, search and warehouse projections, rolling schema upgrades, cross-partition order/payment updates, lease-based workers, and replay-based rebuilds.

Produce a prioritized complete-lens assessment covering authority, consistency, durability, workload/storage, replay/order/idempotency, evolution, replication/partitioning, transactions, derived data, clocks/coordination, processing, tests, observability, repair, and final review. Do not redesign deployment or incident operations except where they affect data correctness.
