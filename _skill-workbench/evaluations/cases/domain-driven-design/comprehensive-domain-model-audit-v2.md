# Comprehensive Domain Model Audit

Perform a comprehensive end-to-end domain-model audit for an insurance platform spanning Sales, Underwriting, Policy Administration, Billing, Claims, and Reinsurance.

The current implementation uses one shared model, contradictory meanings for policy and claim statuses, generic workflow services, persistence-shaped entities, broad transactions, vendor terminology in core code, and no explicit translation between teams. Strategic investment is spread evenly despite Claims being the differentiating capability.

Cover model discovery, expert knowledge, deeper insight, explicit concepts, Ubiquitous Language, scenarios and communication artifacts, context boundaries and relationships, strategic focus, model integrity, tactical meaning, services, Repositories, factories, application and infrastructure boundaries, translation, supple design, tests, refactoring, forbidden patterns, and final review. Do not reduce the answer to implementation mechanics alone.
